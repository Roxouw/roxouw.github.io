# Web Conversion Research

## Overview

Use this skill to turn internet research on conversion patterns into practical decisions for website layout, copy, hierarchy, and mobile behavior.

Default to browsing first. Treat recommendations about sales pages, landing pages, CTA patterns, typography, content density, and persuasive structure as research tasks, not memory tasks.

## Core workflow

1. Identify the page type:
   - homepage
   - landing page
   - pricing page
   - service page
   - proof/case-study section
   - contact section
2. Research current or durable guidance from credible sources before recommending changes.
3. Extract only practical rules that can change the page:
   - layout order
   - headline structure
   - CTA copy and placement
   - trust elements
   - text length
   - typography and readability
   - mobile behavior
4. Translate findings into concrete edits for the project, not generic advice.
5. Prefer small, testable changes over full rewrites unless the evidence strongly supports a structural change.

## Source priorities

Use sources in this order:

1. UX research organizations and usability research
2. Conversion research and experimentation sources
3. Reputable marketing education sources
4. Brand or product examples only as secondary inspiration

Good source types:

- Nielsen Norman Group
- Baymard Institute
- CXL
- HubSpot

Avoid building recommendations from:

- random listicles
- low-signal SEO blogs
- unverified social posts
- aesthetic inspiration without evidence

## What to research

Research only the categories relevant to the task.

### Layout patterns

- best-practice section order
- what should appear above the fold
- how proof and CTA should be positioned
- when to use one-column vs two-column layouts
- how to structure pricing and contact sections

### Sales triggers

- specificity
- urgency
- risk reduction
- social proof
- authority
- clarity of next step
- expectation setting
- friction reduction

### Copy patterns

- headline formulas
- subhead structure
- CTA wording
- proof wording
- objection handling
- amount of text per section
- how much explanation users tolerate before acting

### Readability and typography

- body text density
- line length
- paragraph size
- headings for scanning
- bullets vs paragraphs
- small-screen reading behavior

### Mobile behavior

- first-screen priorities
- CTA visibility on mobile
- section compression
- tap targets
- long-block fatigue
- preview image readability

## Application rules for Rosso

When applying findings to Rosso Labs pages:

- prioritize conversion over decoration
- prefer direct commercial copy over institutional copy
- keep CTA labels objective
- preserve the existing JS hooks, anchors, modal flow, and theme logic
- use research to support sharper hierarchy, not longer pages by default
- if a finding conflicts with the current business goal, favor the business goal and note the tradeoff

## Output format

When using this skill for analysis or proposals, produce:

1. The relevant research-backed findings
2. What they imply for this page
3. The concrete edits to make
4. Any uncertainty or need for testing

Keep outputs practical. Do not dump theory.
